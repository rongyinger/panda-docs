# 如何为你的场景选择合适的 AI 模型

面对几十款大语言模型，很多开发者的第一个问题是：**我到底该用哪个？**

本文从三个核心维度出发，帮你在 10 分钟内找到适合你场景的模型。

## 先问自己三个问题

1. **主要任务是什么？** 对话、代码、长文档分析，还是图片理解？
2. **对延迟有多敏感？** 实时交互 vs 批处理
3. **预算是多少？** 成本约束往往是第一道筛选条件

---

## 维度一：推理与逻辑

需要做数学证明、复杂逻辑推导、多步骤规划的场景，选模型要看"是否能慢下来想清楚"。

| 模型 | 推理评分（相对）| 说明 |
|------|--------------|------|
| o1 / o3 | ⭐⭐⭐⭐⭐ | 专为推理设计，内置思维链，但速度慢 |
| claude-opus-4-7 | ⭐⭐⭐⭐ | 综合推理强，速度可接受 |
| gpt-4o | ⭐⭐⭐⭐ | 均衡，推理略弱于 o1 |
| gemini-2.5-pro | ⭐⭐⭐⭐ | 多模态推理场景有优势 |

**推荐**：数学/代码逻辑 → `o1` 或 `o3-mini`；综合分析 → `claude-opus-4-7`

---

## 维度二：创意写作与内容生成

写故事、营销文案、邮件模板这类任务，关键是文风自然、语言地道。

**Claude 系列在中英文创意写作上普遍优于 GPT 系列**，尤其是长文连贯性更好。GPT-4o 的优势在于格式化输出（表格、列表）更整洁。

**推荐**：`claude-sonnet-4-6` 是性价比最高的选择。

---

## 维度三：代码生成与调试

| 场景 | 推荐 |
|------|------|
| 补全 / 单函数生成 | `gpt-4o-mini` 够用，成本最低 |
| 复杂重构、架构设计 | `claude-sonnet-4-6` 或 `gpt-4o` |
| 算法题 / 竞赛代码 | `o3-mini`（速度快，推理准） |
| 调试（给错误 + 上下文） | `claude-opus-4-7`（上下文理解最强） |

---

## 成本参考（输入/输出，每百万 token，约折算价格）

| 模型 | 输入 | 输出 |
|------|------|------|
| gpt-4o-mini | ¥1.1 | ¥4.4 |
| gpt-4o | ¥17.5 | ¥70 |

| claude-sonnet-4-6 | ¥21 | ¥105 |
| gemini-2.5-flash | ¥0.5 | ¥3.5 |

::: tip 省钱技巧
先用 mini/flash/haiku 原型验证，上线前再测试旗舰版效果提升是否值得溢价。
:::

## 一图总结

```
追求极致效果       → claude-opus-4-7 / o1
推理/数学/代码     → o1 / o3-mini
日常生产首选       → claude-sonnet-4-6 / gpt-4o
超长文档（>100K）  → gemini-2.5-pro
极速低成本         → gpt-4o-mini / claude-haiku-4-5 / gemini-2.5-flash
```
